# Roadmap

## Current State (v1.0.0)
- Core framework architecture established.
- Initial set of 16 canonical skills created.
- Basic template library provided.
- Foundational documentation (Architecture, Lifecycle, Governance) completed.

## Phase 2: Skill Composition Engine
- Implement declarative YAML-based workflow definitions.
- Enable DAG (Directed Acyclic Graph) execution for complex skill chains.
- Add support for conditional branching based on skill outputs.

## Phase 3: Validation Tooling
- Develop a CLI validator for `SKILL.md` structure and schema checks.
- Introduce pre-commit hooks to enforce documentation and quality standards.
- Automate the generation of `SKILL_CATALOG.md` directly from the repository content.

## Phase 4: Community Skills
- Establish a `community/` directory for user-contributed skills.
- Create a promotion pathway from community drafts to canonical skills.
- Define a community governance model.

## Phase 5: Metrics and Analytics
- Build telemetry for tracking skill usage frequency.
- Analyze quality trends over time based on audit findings.
- Measure delivery velocity improvements enabled by the framework.

## Timeline
- **Q3 2026**: Phase 2 delivery.
- **Q4 2026**: Phase 3 validation tooling rollout.
- **Q1 2027**: Phase 4 community integration.
- **Q2 2027**: Phase 5 metrics and analytics dashboard.

## Success Metrics
- **Adoption**: Number of active projects using the framework.
- **Efficiency**: Reduction in manual design/audit overhead time.
- **Quality**: Decrease in production incidents for governed projects.
