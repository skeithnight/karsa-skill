# Contributing

## How to Contribute
We welcome contributions to Karsa Skill! To contribute, follow these steps:
1. Review the existing issues or open a new one to discuss your proposal.
2. Fork the repository.
3. Make your changes and write a descriptive commit message.
4. Submit a Pull Request.

## Creating a New Skill
1. Create a directory in `skills/` using the appropriate naming convention.
2. Add a `SKILL.md` file using the canonical template.
3. Define the YAML frontmatter (name, version, description, category, owner, requires, produces, triggers).
4. Fill out all required markdown sections: Purpose, Execution Workflow, Inputs, Outputs, Success Criteria, Failure Handling, Evidence Requirements.

## Skill Quality Checklist
- [ ] No missing sections.
- [ ] No TBDs or placeholders.
- [ ] At least 3 items per list or table.
- [ ] Valid YAML frontmatter structure.

## Documentation Standards
- Write clearly and professionally.
- Use GitHub Flavored Markdown.
- Ensure cross-references to other documents or skills are valid relative paths.

## Pre-Submission Checklist
- [ ] All new logic verified manually.
- [ ] Cross-references validated to ensure no broken links.
- [ ] Code formatted correctly.

## PR Process
1. Submit the PR with a clear title and description.
2. A maintainer will review the structural validation.
3. Changes may be requested; address them promptly.
4. Once approved, the PR will be merged.

## Code of Conduct
We expect all contributors to maintain a professional, respectful, and inclusive environment. Harassment or abusive behavior will not be tolerated.

## Getting Help
If you need assistance, please open an issue with the "question" label or reach out to the core maintainers.
